Lesson 3 Highlights

Nickson Jean Baptiste

Highlight #1 Version Control

Version control allows for increased file sharing capabilities and the ability to revert major changes to any document pushed to the repository. Each time a document is pushed to the repository it is saved in its exact state in the event of future coding failure. This is essential to web developers in the event they have to go back to a working version of their code after a major change fails or doesn’t yield the desired result.

Highlight #2 Collabrative Work

Collaboration is enhanced with version control because each change made is tracked and linked to individual users. This means improved file sharing and much less redundancy since data is merged. This also allows for groups to pull each other’s files from the repository and start their own branch of documented changes that has no effect on the original document or user.

Highlight #3 Snapshots, not Differences

Since version control systems work on a system of snapshots, each save made to a document that is pushed to a repository creates a snapshot image of the saved file. This is how these systems allow for reverting documents. There is never an overwritten file so every change can be cataloged with each save.

Highlight #4 Version Control Systems

Version control systems notes changes and who makes the changes to a file. Version control systems such as GitHub allows for nondestructive saving in the event a reversion point is needed.

Highlight #5 Repositories

Repositories come in two key form, local and remote. By allowing multiple users to push to a remote repository a user is able to not only track their own work but their coworkers as well. GitHub serves as a remote repository for the purposes of this class and Cloud 9 serves as the local repository in which work is done. Git repositories can be obtained by signing up with GitHub and linking to a local repository.
